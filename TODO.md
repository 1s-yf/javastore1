## 解决方案（逐条对应上方问题）

> 说明：下面是“解决方案/实施步骤”，用于指导后续修复。此次只更新清单文档，不直接改业务代码。

### 1) 打开网页时弹窗提示获取热销商品时产生未知的异常undefined


### 2) 登录首先到首页，然后弹窗提示请登录/注册，点击再跳登录/注册


### 3) 搜索框没有实际搜索功能，返回结果无变化
- **问题确认**
  - 多个页面的搜索表单 `action="search.html"`，但 `search.html` 是纯静态列表，无根据关键词的渲染逻辑，也没有对应的后端“搜索商品”接口。
- **解决方案/实施步骤**
  - 前端：改为 `search.html?keyword=...`，并在 `search.html` 内用 JS 读取 URL 参数，调用搜索接口拉取数据渲染列表。
  - 后端：新增 `GET /products/search?keyword=`（可支持分页），在 `ProductMapper.xml` 新增 like 查询。
  - 白名单：如需匿名搜索，也要把 `/web/search.html`、`/products/search` 加入白名单。

### 4) 秒杀/优惠券/电脑VIP/外卖/超市点击没有反应
- **问题确认**：`web/*.html` 中该些链接多为 `href="#"`。
- **解决方案**
  - 未开发功能：改成禁用态（置灰、不可点）并给出“敬请期待”提示，避免“没反应”的体感。
  - 已开发功能：增加对应页面或后端路由。

### 5) 左侧分类内容与电脑商城无关，且子分类点击无反应
- **问题确认**
  - `src/main/resources/static/js/menu.js`
    `typelist` 大量分类是京东风格的通用分类，且代码主要做“悬停展示”，没有点击后的导航逻辑。
- **解决方案**
  - 用你实际的电脑类目重新设计 `typelist`（可以只保留“笔记本/台式机/显示器/外设”等几类），或者改为从后端拉取分类树。
  - 为 `.second-menu-li`、`.third-menu-li` 增加 `click`事件，点击跳转到 `search.html?categoryId=...`（或按 name 搜索）。

### 6) “新到好货”点不同商品进去都是同一个商品内容


### 7) 页脚版权信息和二维码需替换
- **问题确认**
  - 多个页面都写了同样的版权文案。
- **解决方案**
  - 如果你不用模板引擎：找出所有 `web/*.html` 页面的 footer 部分，统一替换版权文案和二维码图片。
  - 如果你要节约维护成本：建议将 footer 抽成一个公共模板并通过引入方式复用（后续可以考虑 Thymeleaf）。

### 8) 页脚的“卖家帮助/联系我们”等链接点击无正确跳转
- **解决方案**
  - 当前是 `href="#"`，最低成本方案：
    - 给这些链接绑定 click，弹出“敬请期待”。
    - 或者指向一个 `help.html/about.html`的说明页。

### 9) 购物车/收藏的数量加减无法同步数据库，数量=0不自动删除
- **问题确认**
  - `static/js/cart.js`只是 DOM 上改数字，而且总价计算 `calcTotal()` 被注释。
  - `favorites.js`/`search.js` 只做了心形图标的 toggle，不保存。
  - 后端暂无 Cart/Favorite 相关 Controller/Service/Mapper。
- **解决方案**
  - 先实现后端模块：`CartController`、`CartService`、`CartMapper`，针对 `t_cart`定义 CRUD（列表、加入、改数量、删除、批量删除）。
  - 前端：
    - `cart.html` 页面加载时 AJAX 获取购物车列表并渲染
    - `+/-/删除` 调用后端接口更新，成功后刷新单行小计与总价
    - 数量减到 0 时可直接调用删除接口并删除该行
  - 收藏：如果需持久化，建议新增收藏表 + 后端接口，前端调用接口而不是 toggle。

### 10) 订单页面逻辑不闭环，按钮无效，部分功能页（头像/收货管理）报错
- **问题确认**
  - `orders.html`、`orderConfirm.html`、`payment.html` 大部分是静态演示（或者只是页面跳转），未调用后端创建订单/支付/确认收货等接口，后端也暂无订单模块。
  - `upload.html` 使用 `$.cookie("avatar")`，但页面没有引入 `jquery.cookie.js`，可能报错导致交互中断。
- **解决方案**
  - 先定义订单状态流转：待付款 -> 已支付 -> 已发货 -> 已签收 -> 完成/售后
  - 补后端模块：`OrderController/OrderService/OrderMapper`，对应 `t_order`、`t_order_item`
  - 前端：
    - `orderConfirm.html`：点“在线支付”时调用“创建订单”接口，获得 `orderId`
    - `payment.html`：点“确认付款”时调用“支付”接口（项目内可以先做“模拟支付”更新状态）
    - `orders.html`：列表页从后端拉取订单列表渲染，筛选条件（待付款/待收货等）走参数或状态
    - “确认收货”：调用“签收”接口更新状态
  - 修头像页：在 `upload.html` 页头引入 `../bootstrap3/js/jquery.cookie.js`（参考 `login.html`的引入方式）。

### 11) 设置密码要有等保护限制，不明文存储
- **问题确认**
  - 后端已经使用了“加盐 + MD5”（`UserServiceImpl.getMD5Password()`），但前端密码输入框是 `type="text"`，且没有校验新密码复杂度。
- **解决方案**
  - 前端：
    - 将密码输入框改为 `type="password"`
    - 增加复杂度校验：长度>=8，包含大写、小写、数字、特殊字符，且“新密码”与“确认密码”一致
  - 后端：
    - `change_password` 接口内也要做复杂度校验（避免绕过前端）
  - 未来安全升级：可以考虑用 BCrypt 替代 MD5。

### 13) [新增]未登录时首页调用商品接口被拦截，导致弹窗报错（非 JSON）
- **解决方案**
  - `LoginInterceptorConfigurer` 白名单加入（二选一）：
    - 仅允许匿名的商品相关接口：`/products/**`
    - 或者允许匿名管理页和请求页面：`/web/index.html`不发起需登录的 AJAX
  - 前端全局处理 AJAX 返回为 HTML 的情况，统一引导到登录页。

### 14) [新增]“自动登录”未生效，且密码明文存 cookie
- **问题确认**
  - `static/js/autoLogin.js` 绑定的是 `#loginBtn`，但实际登录页按钮是 `#btn-login`，所以点“登录”时不会写入 cookie。
  - 另外，明文把密码放在 cookie 风险非常高（泄露后等于账号被直接控制）。
- **解决方案**
  - 把绑定按钮 id 改为 `#btn-login`（或者页面统一 id）。
  - 如果保留自动登录：改为后端签发安全 token（或服务端持久化登录态），不在前端存密码。

### 15) [新增]头像上传页未引入 cookie 插件，导致跑出 JS 报错
- **解决方案**
  - 参考 `web/login.html` 的引入方式，在 `web/upload.html` 中引入：`../bootstrap3/js/jquery.cookie.js`
  - 页面加载时先判空：如果 cookie 中没有 `avatar`，保持默认头像不要强行 `attr("src", undefined)`。
